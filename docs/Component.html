Class: Component

Component

Class representing a component, which is an object whose responsibilities are:
  • Render the HTML results of a provided renderer function inside a specified element of the showing document, making as less DOM changes as possible.
  • Manage nested child components. Child components are components which render in an element inside this component. When this Component re-renders, it restores its child's subtrees on their places. Child Components can be added manually (See Component#addChildComponent) or created dynamically by this Component via custom tag elements (See Component#childTags).
  • Manage event listeners, restoring them each re-rendering.
Components render when you call start(), and updates each time you call the render() function.

var counter = 1;
var component = new Component(
 () => '<div>Counter: <span>'+counter+'</span></div>', // renderer function
 'mycomponent' // HTML element id
 );
component.start(); // first render
setInterval(() => {
   counter++; 
   component.render(); // component re-render
}, 1000);

createChildComponent(tagName, childTagElement, id) → {Component}

Creates a new Component for a specified tag name to be placed in the place of a given HTML element. This method is intended to be overrided by subclasses. By default (if not overriden), this method searches for a class with the same name as the tagName and instantiates an object of this class passing the id to it. In this sense, a parent Component specifying a child tag name 'ChildComponent' as this: 
new Component(renderer, 'parentId', ['ChildComponent'])
will create instances of ChildComponent in all places where the tag <childcomponent> is found in the HTML provided by the parent rendereer function.
Parameters:

render()

Render this Component, which consists in:
  1. Save the child Component DOM trees, because they may be moved to another place in the DOM.
  2. Call the renderer function.
  3. Calculate the differences between the previous "virtual" DOM of this Component and the new "virtual" DOM provided by the renderer function, skipping those elements where child nodes are rendering.
  4. Patch the real DOM with the previously computed differences.
  5. Patch the previous "virtual" DOM with the previously computed differences, and save it as the next previous "virtual" DOM.
  6. Restore the child Components in their new places if they where moved to another part in the DOM.
  7. Restore event listeners in their corresponding elements, because some could be moved to another place in the DOM.
  8. Create child nodes if new elements with tag name in Component#childTags are found in the HTML.
